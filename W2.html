Two New Functions

I showed you how the while and for functions worked earlier in the workshop, bt here is a little more information on them which you might find useful when designing your game.

  1. The function for performs an operation a number of times; the number of times has to be specified. We specify through the use of a counter, often but not necessarily denoted i. We refer to this process as “a for loop”. If we try:

    a<-0

    for(i in 1:10){a<-a+i}

    we get a value a, initially equal to 0, but then has the values 1, 2, 3,... 10 added to it, for a total of 55. In contrast, if we try

    a<-0

    for(i in 1:10){a<-a+1}

    we’ll get a value for a of 10. This is because i is not contained in the for loop. R still knows how many times you want the loop to run for, though, so it still does what you ask it to 10 times.

  2. The while function has some similarity with the for function, in that it runs loops for us. The big difference is that the for function needs you to specify in advance how many loops to run. The while function will keep running loops so long as a given property holds.

    So for instance, we can write a function that tells us how many times we can divide an integer by 2, as we see on the next page.

    divideby2<-function(x){

    count<- -1

    status<-1

    while(status==1)

    {count<-count+1

    if(x%%2!=0){(status<-0)}

    x<-x/2

    }

    return(count)

    }

    Here, the while loop keeps dividing the input by 2, until such time as that doesn’t give us a whole number. At that point the program exits the while loop. By adding one to the count each time the while loop runs, we learn how many times dividing by two was possible.
